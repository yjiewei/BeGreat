

# 牛客网上的面试经验分享

```
1. 前后端通信是不是问swagger？
答：不是，swagger 只是指接口之间的约定。 具体来说前后端通信一般分为两种吧， restful（基础，答不出这个直接挂了，面试官肯定也准备了一些题目顺势接着往下问） ，graphql （加分项，不清楚面试官掌握到哪种程度，可能也希望和面试者可以一起探讨下，面评++），当然如果能接着谈 grpc 啥的也好。
GraphQL 是一种针对 Graph（图状数据）进行查询特别有优势的 Query Language（查询语言），所以叫做 GraphQL。

2. 啥是鉴权，怎么鉴，通过什么方式
答：举例来说，服务端的某个接口需要验证当前访问的用户是否有这个权限。 我这里假设前端在第一次登陆后，服务端给前端种了一个 userId 的 token 放在 cookie 中。之后前端请求服务端的时候，总是会带上这个 cookie。服务端则可能会在需要权限控制的接口配置一个 guard，guard 里面会利用一个只保存在服务端的密钥 + userId 的 token 计算出 userId，然后去数据库中查这个 userId 对应的权限列表，并与这个接口静态配置的权限进行匹配，看是否符合，如果没错，则服务端的接口层通过继续，不通过就 403，401 之类的了，这就是鉴权。 这里只是举了很简单的例子，具体怎么做则不一定，看你实际项目中是如何采用的。

3. 怎么分析一条SQL语句执行性能
答：没错，就是 explain。面试官应该是想问你具体哪些字段往往比较有用。你有没有碰到一些要优化的情况，然后自己通过 explain 分析解决了。

```
